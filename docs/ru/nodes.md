# Типы узлов (Node Types)

Axolotl поддерживает пять типов узлов в пайплайне. Каждый тип имеет свою роль и параметры конфигурации.

## Source (Receive)

Собирает ввод для пайплайна. Поддерживает четыре режима:

| Режим | Описание |
|-------|----------|
| `text` | Прямой текстовый ввод |
| `file` | Ссылка на файл — читает файл относительно `targetPath` |
| `url` | URL — загружает содержимое по URL |
| `project` | Директория проекта — список файлов в `targetPath` |

### Конфигурация

| Поле | Тип | Описание |
|------|-----|----------|
| `sourceType` | `"text" \| "file" \| "url" \| "project"` | Режим источника |
| `sourceData` | `string` | Текстовое содержимое (text mode) |
| `filePath` | `string` | Путь к файлу (file mode) |
| `url` | `string` | URL для загрузки (url mode) |
| `projectPath` | `string` | Путь к директории (project mode) |
| `maxDepth` | `number` | Максимальная глубина директории |
| `maxFiles` | `number` | Максимальное количество файлов |

## Agent

LLM-узел с инструментами для генерации кода.

### Инструменты

| Инструмент | Описание |
|------------|----------|
| `file_write` | Запись файла в `targetPath` |
| `file_read` | Чтение файла |
| `directory_read` | Список содержимого директории |
| `bash` | Выполнение bash-команды |
| `web` | Загрузка URL |
| `grep` | Поиск в содержимом файлов |
| `glob` | Поиск файлов по шаблону |

### Конфигурация

| Поле | Тип | Описание |
|------|-----|----------|
| `model` | `string` | Имя LLM-модели |
| `systemPrompt` | `string` | Системный промпт |
| `userPrompt` | `string` | Пользовательский промпт (поддерживает `{{sourceData}}`) |
| `agentType` | `"coder" \| "reviewer" \| "architect"` | Роль агента |
| `enabledTools` | `string[]` | Список включённых инструментов |
| `maxTokens` | `number` | Максимум токенов в ответе |
| `temperature` | `number` | Температура LLM (0-2) |

## Review

Генерирует план разработки и выполняет проверки качества перед генерацией кода.

### Проверки

| Проверка | Описание |
|----------|----------|
| **Premortem** | Предвидит возможные проблемы и предлагает меры |
| **Prism** | Анализирует план с разных архитектурных перспектив |
| **Postmortem** | Оценивает завершённый план на пробелы и улучшения |

### Режимы

| Режим | Поведение |
|-------|-----------|
| `manual` | Всегда показывает диалог утверждения |
| `auto` | Автоисправление до `maxIterations` (по умолч. 3), после ошибка |
| `hybrid` | Автоисправление до `maxAutoIterations`, затем диалог с человеком |

### Конфигурация

| Поле | Тип | Описание |
|------|-----|----------|
| `premortem` | `boolean` | Включить premortem |
| `prism` | `boolean` | Включить prism |
| `postmortem` | `boolean` | Включить postmortem |
| `mode` | `"manual" \| "auto" \| "hybrid"` | Режим review |
| `maxAutoIterations` | `number` | Максимум автоисправлений (hybrid) |
| `maxIterations` | `number` | Максимум итераций (auto) |
| `generatePlan` | `boolean` | Генерировать план разработки |

## Verifier

Валидирует сгенерированный код по настраиваемым критериям.

### Проверки

| Проверка | Описание |
|----------|----------|
| `syntaxCheck` | Проверка синтаксиса для обнаруженного языка |
| `requiredPatterns` | Проверка обязательных паттернов (regex) |
| `testCommand` | Запуск тестовой команды |
| `maxFileSizeKb` | Отклонение файлов превышающих лимит |

### Конфигурация

| Поле | Тип | Описание |
|------|-----|----------|
| `checks` | `object` | Включённые проверки с параметрами |
| `rewriteOnFail` | `boolean` | Автоперезапись при неудаче проверки |
| `maxRewriteRetries` | `number` | Максимум попыток перезаписи (по умолч. 3) |
| `validationCriteria` | `string` | Пользовательские критерии валидации |

Вердикт verifier:

```json
{
  "status": "PASS",
  "checks": [
    { "name": "syntax", "passed": true },
    { "name": "size", "passed": true }
  ],
  "summary": "All checks passed"
}
```

## Output

Собирает и выводит результаты выполнения.

### Режимы

| Режим | Описание |
|-------|----------|
| `stdout` | Вывод в консоль |
| `log` | Запись в лог приложения |
| `summary_report` | Генерация `pipeline-report.md` в `targetPath` |

### Конфигурация

| Поле | Тип | Описание |
|------|-----|----------|
| `mode` | `"stdout" \| "log" \| "summary_report"` | Режим вывода |
| `reportPath` | `string` | Путь к файлу отчёта (summary_report) |
| `includeReview` | `boolean` | Включить review в отчёт |
| `includeFiles` | `boolean` | Включить список файлов |
| `includeVerification` | `boolean` | Включить результаты проверки |
| `includeMetrics` | `boolean` | Включить метрики выполнения |
