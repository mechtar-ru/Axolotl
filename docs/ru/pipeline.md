# Pipeline System

Pipeline — это основной движок выполнения Axolotl. Он оркестрирует многоэтапные рабочие процессы генерации и проверки кода.

## Обзор

Pipeline — это последовательность этапов с явным порядком зависимостей. Этапы выполняются через топологическую сортировку: этапы одного уровня зависимостей выполняются параллельно; этап начинается только после завершения всех его зависимостей.

```
        ┌──────────┐
        │ Receive  │
        └────┬─────┘
             │
        ┌────▼─────┐
        │  Review   │◄──── Шлюз утверждения человеком
        └────┬─────┘
             │
        ┌────▼─────┐
        │  Agent    │
        └────┬─────┘
             │
        ┌────▼─────┐
        │  Verify   │
        └────┬─────┘
             │
        ┌────▼─────┐
        │  Output   │
        └──────────┘
```

## Пайплайн по умолчанию

Quick Start создаёт 5-этапный пайплайн:

1. **Receive** — сбор исходного ввода (текст, файл, URL, или директория проекта)
2. **Review** — генерация плана разработки, проверки premortem/prism/postmortem
3. **Agent** — LLM с инструментами, пишет код через file_write/read/bash
4. **Verify** — синтаксические проверки, тестовые команды, контроль качества
5. **Output** — генерация отчёта (или stdout/log)

## Типы этапов

| Тип | Описание |
|-----|----------|
| `receive` | Сбор ввода — текст, файл, URL, или директория проекта |
| `review` | Генерация плана с тремя проверками. Всегда требует утверждения человеком |
| `agent` | LLM с инструментами (file_write, file_read, directory_read, bash, web, grep) |
| `verifier` | Валидация — синтаксис, паттерны, тестовые команды, размер файлов |
| `output` | Генерация отчёта — режимы: `stdout`, `log`, `summary_report` |

## Поток выполнения

### Build

```bash
POST /api/schemas/{schemaId}/pipeline/build
```

Генерирует узлы пайплайна из определений этапов схемы. Создаёт Vue Flow узлы и связи, готовые к выполнению.

### Execute

```bash
POST /api/schemas/{schemaId}/pipeline/execute
```

1. Создаёт `ExecutionRun` со статусом `running`
2. Выполняет топологическую сортировку этапов
3. Запускает этапы уровень за уровнем (параллельно внутри уровня)
4. Приостанавливается на `AWAITING_APPROVAL` (review-узлы)
5. Возобновляется после утверждения человеком

### Status

```bash
GET /api/schemas/{schemaId}/pipeline/status
```

Возвращает текущий статус пайплайна, состояния этапов и выводы этапов.

### Retry

```bash
POST /api/schemas/{schemaId}/pipeline/retry
```

Создаёт дочерний `ExecutionRun`, возобновляющий выполнение с первого упавшего этапа. Очищает только упавшие и зависимые этапы.

### Cancel

```bash
POST /api/schemas/{schemaId}/pipeline/cancel
```

Устанавливает флаг отмены; выполняющиеся этапы обнаруживают его и останавливаются. Записывает статус `cancelled`.

## Утверждение Review

Review-узлы всегда приостанавливают пайплайн и показывают диалог утверждения с сгенерированным планом.

Диалог предлагает:
- Сгенерированный план разработки
- **Accept** — утверждает и продолжает выполнение
- **Reject** — помечает узел как упавший

Пайплайн ожидает бесконечно на этапе review, пока пользователь не ответит.

API эндпоинты:

```bash
POST /api/execution/{executionId}/approve-review?nodeId={nodeId}
POST /api/execution/{executionId}/reject?nodeId={nodeId}
```

## Выводы этапов

Каждый этап производит выводы, которые могут быть потреблены downstream-этапами через `inputMapping`:

| Этап | Ключ вывода | Описание |
|------|-------------|----------|
| Receive | `sourceContent` | Содержимое ввода |
| Review | `plan` | Сгенерированный план разработки |
| Agent | `generatedFiles` | Список созданных/изменённых файлов |
| Verify | `verdict` | JSON с `status` (PASS/FAIL) и `checks` |
| Output | `report` | Сгенерированный отчёт |

Межэтапные ссылки используют dot-нотацию: <code v-pre>{{agent.generatedFiles}}</code>.

## Реализация бэкенда

Ключевые классы:

- `PipelineService.java` — оркестрация выполнения, build, retry, cancel, resume
- `Pipeline.java` — модель с этапами, флагом TDD mode
- `Stage.java` — модель с именем, типом, зависимостями, маппингами ввода

Pipeline service использует `CompletableFuture` для выполнения этапов с таймаутом 5 минут на этап. Этапы выполняются на virtual threads.

## Реализация фронтенда

Ключевые компоненты:

- `PipelinePanel.vue` — боковая панель с уровнями этапов и индикаторами статуса
- `schemaStore.ts` — Pinia store с состоянием пайплайна (статус, этапы, выводы)
- `StudioView.vue` — основная вью с раскладной панелью пайплайна

## Конфигурация

Модель пайплайна по умолчанию — `deepseek-v4-flash-free` (доступна через бесплатный Zen API). Может быть настроена по этапу в PipelinePanel.
