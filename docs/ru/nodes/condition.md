# Узел Condition

Узел **Condition** создаёт логику ветвления в вашем workflow на основе оценки входных данных.

## Обзор

```
┌─────────────────┐
│    Condition    │
│                 │
│ if {{input}}    │
│   содержит     │
│ "ошибка"?       │
│                 │
│ 🔽 Да    🔽 Нет
└─────────────────┘
```

## Настройка

| Свойство | Тип | Описание |
|----------|------|----------|
| **name** | string | Имя узла |
| **condition** | string | Выражение условия |
| **conditionType** | string | contains, equals, matches, custom |

## Типы условий

| Тип | Описание | Пример |
|------|----------|---------|
| **contains** | Проверка наличия подстроки | `"ошибка"` в `{{input}}` |
| **equals** | Точное совпадение | `{{input}}` == `"да"` |
| **matches** | Регулярное выражение | `{{input}}` matches `\d+` |
| **custom** | JavaScript выражение | `{{input}}.length > 10` |

## Использование

1. Добавьте узел Condition
2. Подключите вход от предыдущего узла
3. Напишите выражение условия
4. Подключите выход **Да** для выполнения условия
5. Подключите выход **Нет** если условие не выполнено

## Пример

```javascript
// Условие: Проверить наличие "срочно"
{{input}} contains "срочно"

// Если да → Отправить уведомление
// Если нет → Отправить стандартное сообщение
```

## Несколько условий

Для сложной логики, свяжите несколько узлов Condition:

```
Source → Agent → Condition
                  ├─ да → Узел уведомления
                  └─ нет → Обычный узел
```

## Обработка ошибок

Используйте Condition для обнаружения ошибок:

```javascript
// Проверить наличие ошибок
{{input}} contains "ошибка"

├─ да → Узел Fallback
└─ нет → Продолжить обработку
```
