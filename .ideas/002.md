You are an experienced mobile developer and product designer. Your task is to design and implement a **Russian‑language analog of the How We Feel app** – a native mobile app for iOS and Android that lets users track their emotions, keep a mood diary, analyze dynamics, and receive psychological insights.

## Functional Requirements

1. **Registration & Login**  
   - Simple authentication (email + password) or via social logins (Google, VK ID, Apple ID).  
   - Guest mode (store data locally on device without an account).

2. **Emotion Tracking**  
   - Main screen: a grid of 4–6 basic emotions, each with its own colour (adapted for Russian culture).  
     - **Green** – calm, joy, interest  
     - **Blue** – sadness, melancholy, longing  
     - **Red** – anger, irritation, rage  
     - **Yellow** – anxiety, fear, worry  
     - **Purple** – pleasure, delight, inspiration  
     - **Grey** – apathy, fatigue, indifference  
   - After selecting a colour, a detailed list of specific feelings (at least 12 in Russian, including words like "умиление" (tenderness), "авось" (hopeful uncertainty), "неловкость" (awkwardness), "злорадство" (schadenfreude), "стыд" (shame)).  
   - Ability to add a note (up to 500 characters), context (where, with whom, activity), and intensity (1–5 scale).

3. **History & Analytics**  
   - Calendar view with colour‑coded emotion indicators per day.  
   - Graphs (line or bar) showing mood trends for week/month/year.  
   - Statistics: most frequent emotions, triggers (based on keywords from notes).  
   - Export data to JSON or CSV.

4. **Tips & Support**  
   - For each basic emotion: a short, actionable recommendation in Russian (breathing exercise, physical activity, cognitive reframing, etc.).  
   - “Resources” section: links to Russian‑language articles about emotion regulation, contacts for psychological help (customizable).

5. **Design & UX (Mobile)**  
   - Minimalistic, calm interface optimised for touch.  
   - Dark and light theme (switch).  
   - Smooth animations when selecting emotions and switching days.  
   - Native navigation (tab bar or bottom navigation).

6. **Technical Requirements (iOS & Android)**  
   - **Platform**: Native (Swift/SwiftUI for iOS, Kotlin/Jetpack Compose for Android) **or** cross‑platform framework (React Native, Flutter, or .NET MAUI). Choose the most efficient approach and justify it.  
   - **Local storage**: offline‑first – store all entries in a local database (SQLite, Room, Core Data, or Realm).  
   - **Optional backend**: Firebase (Firestore + Auth) or a custom Node.js backend for syncing across devices and backup. If you include sync, handle conflicts gracefully.  
   - **Offline support**: full functionality without internet; sync when connection is restored.  
   - **Push notifications** (optional): daily reminders to log emotions.  
   - **Privacy & compliance**: users can delete all their data locally and remotely (if backend used). Comply with Russian 152‑FZ (notice of data collection, ability to delete account).

7. **Localisation**  
   - Fully in Russian language.  
   - Russian date formats (DD.MM.YYYY).  
   - Culturally adapted emotion vocabulary and design patterns (avoid Western‑centric defaults).

## What to Provide

1. **Detailed technical specification**  
   - App architecture (MVP, MVVM, or MVI).  
   - List of screens and navigation flow.  
   - Data models (emotion entry, user settings).  

2. **Complete code** for the chosen platform(s)  
   - All major screens: Emotion Grid, Entry Details, Calendar/Analytics, Settings, Tips.  
   - Local database schema and migration logic.  
   - (Optional) Backend code if using a custom server (Node.js + Express) or Firebase security rules.  

3. **Build & deployment instructions**  
   - How to run the app on an emulator/physical device.  
   - Steps to build a release version for iOS (TestFlight/App Store) and Android (Google Play).  
   - Environment setup (API keys, signing certificates).  

4. **Example UI layouts** (text description or pseudo‑code) with Russian labels and colour palettes.

## Additional Desires

- **Simple AI helper** (on‑device or lightweight backend): suggest similar past emotions based on note embeddings.  
- **Code should be clean**, well‑commented (English or Russian comments allowed).  
- **Error handling** and loading states for async operations (e.g., saving entries, syncing).  
- **Accessibility** support (VoiceOver/TalkBack).
