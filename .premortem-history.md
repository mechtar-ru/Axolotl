# Premortem: Axolotl Systemic Risks

## 2026-06-17 — Full System Premortem

### Scoring
- **P(робь)**: Probability of failure (Low/Medium/High)
- **S(everity)**: Impact (L1=minor annoyance, L3=partial outage, L5=data loss / full outage)
- **R(isk)**: P × S (L=low, M=medium, H=high, C=critical)

---

## CRITICAL (R=H-C)

### [FIXED] C1. ForkJoinPool thread leak on timeout (b2df762d)
**Where**: `NodeRouter.executeNode()` lines 140-240
**Mechanism**: `CompletableFuture.supplyAsync()` dispatches strategy call into `ForkJoinPool.commonPool()`. `.get(timeoutSecs, ...)` threw `TimeoutException` but **never called `future.cancel(true)`**.
**Impact**: After N timeouts (N = CPUs - 1), ForkJoinPool exhausted. JVM restart needed.
**Fix**: Extracted future as local variable, added `future.cancel(true)` in TimeoutException catch block.

### [FIXED] C2. Class-level @Transactional on SchemaExecutionService (b2df762d)
**Where**: `SchemaExecutionService.java` line 25 — `@Transactional` on class
**Mechanism**: All public methods including `executeSchema()` (10+ min pipelines) wrapped in Neo4j transactions with 60s default timeout. Every FLUTTER pipeline hit `TransactionTimeoutException`.
**Fix**: Removed `@Transactional` from class. Added `@Transactional(readOnly = true)` on 7 fast getter methods only.

## HIGH (R=M-H)

### [FIXED] H1. Null tool_call_id in agent responses (b2df762d)
**Where**: `ToolCallParser.java` — JSON parsing of LLM tool_calls
**Mechanism**: Weak models return `tool_calls` with `id=null`. Without null-id fallback, remaining tools unexecuted.
**Fix**: Added `putIfAbsent("id", "call_"+calls.size())` in ToolCallParser.parse() — every parsed tool call gets a non-null generated ID.

### [FIXED] H2. OpenRouter free tier hangs (b2df762d)
**Where**: `CustomLlmProvider.java` — default TIMEOUT_SECONDS
**Mechanism**: OpenRouter free-tier proxy keeps HTTP connections open for >120s. 3 retries × 120s = 6 min per planner call blocks ForkJoinPool thread.
**Fix**: Reduced `TIMEOUT_SECONDS` from 3600 to 120. Existing `.timeout(Duration.ofSeconds(120))` on raw HTTP path unchanged.

### [FIXED] H3. Plugin bridge zombie leaks (b2df762d)
**Where**: `PluginLifecycleManager.initialize()` — Bun subprocess lifecycle
**Mechanism**: JVM crash leaves orphan Bun processes. Each ~100MB RSS. Overtime OOM.
**Fix**: Added `pkill -f plugin-bridge.js` at start of `initialize()`. Also in @PreDestroy shutdown.

### [FIXED] H4. Neo4j connection pool exhauston (b2df762d)
**Where**: `application.yml` — no explicit pool config
**Mechanism**: No connection pool limits. Concurrent schema executions + graph loading exhaust default.
**Fix**: Added explicit pool: max-active:16, max-idle:4, max-lifetime:300000ms, connection-timeout:5000ms.

## MEDIUM (R=M)

### [FIXED] M1. generatedFilesRegistry is in-memory, lost on restart (b2df762d)
**Where**: `PipelineStageExecutionService.onPipelineComplete()` + `ExecutionRepository.updateRunGeneratedFiles()`
**Mechanism**: generatedFilesRegistry was never persisted to Neo4j. Over JVM restart, multi-session lost file tracking.
**Fix**: Extracted `extractGeneratedFiles()` helper in PipelineStageExecutionService. Called `executionRepository.updateRunGeneratedFiles()` before `stateManager.removeSchema()`.

### [FIXED] M2. Hardcoded allowedCommands list (b2df762d)
**Where**: `ToolHandlerService.java` — static reference to `ToolExecutorImpl.DEFAULT_ALLOWED_COMMANDS`
**Mechanism**: Bash command whitelist was hardcoded in Java. Adding commands required recompilation.
**Fix**: Added `@Value("${axolotl.tools.allowed-commands:}")` to ToolHandlerService with `getAllowedCommands()` helper. Falls back to static set when config is empty.

### M3. WebSocket message buffer overflow
**Where**: `ExecutionWebSocketHandler.java`
**Mechanism**: Agent streaming tokens send individual `sendToken()` messages at high frequency. During long-running agent execution (especially with streaming), the outbound message buffer can grow unbounded. No flow control or backpressure.
**Impact**: WebSocket session memory grows linearly with agent output length. On constrained environments, OOM.
**P**: Medium (long outputs from 14B+ models)
**S**: L2 (memory pressure, possible WS disconnect)
**R**: M
**Fix**: Batch tokens or use fixed-size ring buffer for WebSocket messages.

### M4. No frontend test coverage
**Where**: `frontend/src/` — 0 tests across all Vue components
**Mechanism**: StudioView (889L), QuickStartDialog (811L), TimelineView (880L), DashboardView (1150L), SessionChatDialog (280L) — none have tests. Refactoring these is blind.
**Impact**: Frontend regressions go undetected. Every frontend change is a manual verification risk.
**P**: High (active frontend development)
**S**: L1 (regressions caught late, slow dev cycle)
**R**: M
**Fix**: Add Vitest for composables at minimum (`useExecution`, `useReview`).

## LOW (R=L)

### L1. Multiple schemaRepository.findById() calls within same method
**Where**: Several strategies call `schemaRepository.findById(schemaId)` multiple times in the same method (e.g., AgentNodeStrategy before the recent fix had 2 calls). Each call is a separate Neo4j query.
**Impact**: 2-4 redundant Neo4j queries per node execution. Adds ~20-40ms overhead.
**P**: High (always happens)
**S**: L1 (minor latency)
**Fix**: Cache locally at method start.

### L2. Pipeline retry duplicates stage execution history
**Where**: `PipelineStageExecutionService.retryPipeline()`
**Mechanism**: Retrying a pipeline creates new ExecutionRun, but the previous run's stage outputs remain visible in the UI. User sees duplicate "Session N" entries.
**Impact**: Timeline gets cluttered with failed+retry pairs.
**P**: High (retry is a common workflow)
**S**: L1 (cosmetic)
**Fix**: Hide retry child runs from timeline, or merge into parent.

### L3. No LLM cost tracking per-user
**Impact**: Cannot bill per tenant.
**P**: Low
**S**: L1
**Fix**: Add tenant ID to LlmUsage record.
