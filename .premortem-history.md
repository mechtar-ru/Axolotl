## CRITICAL (R=H-C)

### C3. ForkJoinPool.commonPool() for blocking I/O — SYSTEMIC
**Where**: `NodeRouter.java:139` (CompletableFuture.supplyAsync default pool)
**Mechanism**: Every node execution (agent, review, verifier, output) dispatches into `ForkJoinPool.commonPool()` — the JVM-wide pool shared with parallel streams, other CFs, Spring internals. All node operations are blocking (LLM HTTP calls, `Thread.sleep`, `Process.waitFor`, file I/O).
**Impact**: With 3+ concurrent node executions (parallel pipeline stages, 2 concurrent schemas), all commonPool threads (`availableProcessors - 1`, ~7) block on I/O. System-wide degradation: parallel streams hang, other CF-based ops deadlock, new executions queue indefinitely.
**P**: High (every node execution hits this)
**S**: L4 (cascading system-wide degradation)
**R**: **C**
**Fix**: Replace with `Executors.newVirtualThreadPerTaskExecutor()` — virtual threads are ideal for blocking I/O. Inject as shared bean.

### C4. Zombie process accumulation (Flutter/build tools) — SYSTEMIC
**Where**: `AgentNodeStrategy.java:639-672`, `FlutterScaffoldHelper.java:39-105`, `FixPassOrchestrator.java:59-109`
**Mechanism**: Every `ProcessBuilder.start()` call uses `p.waitFor(timeout, unit)` but **never calls `p.destroy()` on timeout** (~10 sites). Additionally, process stdout/stderr is **never consumed** before `waitFor()` — if OS pipe buffer (64KB) fills, process hangs permanently.
**Impact**: Each runaway process accumulates: ~10 processes per pipeline run (flutter pub add ×6, pub get, build, analyze). File descriptor exhaustion, PID exhaustion, disk pressure from build artifacts, memory from Dart compiler processes.
**P**: High (any slow network or heavy build triggers this)
**S**: L4 (OS resource exhaustion, manual kill required)
**R**: **C**
**Fix**: Extract `SafeProcess` helper: `try { p.waitFor(...) } finally { if(p.isAlive()) p.destroyForcibly(); }`. Drain streams via thread or `ProcessBuilder.Redirect.DISCARD`.

### C5. graph_query tool allows arbitrary destructive Cypher
**Where**: `ToolHandlerService.java:486-518` (handleGraphQuery)
**Mechanism**: `session.run(query)` executes raw Cypher from LLM tool call. No validation, no read-only enforcement. Agent prompt injection → arbitrary `CREATE/DELETE/MERGE/DETACH DELETE` on the database.
**Impact**: Complete database compromise — data deletion, exfiltration, schema destruction. Prompt injection on agent node → attacker runs MATCH (n) DETACH DELETE n.
**P**: High (tool is documented and active)
**S**: L5 (complete data loss)
**R**: **C**
**Fix**: Wrap in `session.executeRead(...)` or add query validator rejecting `CREATE/SET/DELETE/MERGE/DROP/CREATE INDEX`.

### C6. findByUserId() returns empty — node props never saved
**Where**: `Neo4jSchemaRepository.java:51` (`MATCH (s:WorkflowSchema {userId: $userId})`)
**Mechanism**: `save()` MERGE only sets `s.data = $data`. Node properties (`userId`, `name`, `createdAt`, etc.) are NEVER populated. Query filters on `s.userId` → always finds zero results.
**Impact**: `getSchemasByUserId()` → empty. User cannot list their schemas. Schema listing by user is completely broken.
**P**: Certain (every call returns empty)
**S**: L4 (core feature broken)
**R**: **C**
**Fix**: Add all node properties to MERGE SET clause, or change query to filter JSON blob.

### C7. findByStatus() references unbound $schemaId parameter
**Where**: `Neo4jExecutionRunRepository.java:15-21`
**Mechanism**: `@Query("MATCH (r:ExecutionRun {schemaId: $schemaId, status: $status}) ...")` — `$schemaId` is never bound in method signature. Will throw `ParameterMissingException` on every call.
**Impact**: Any code path calling `findByStatus()` crashes with runtime exception. Execution lookup is broken.
**P**: Certain
**S**: L4 (runtime crash on every call)
**R**: **C**
**Fix**: Remove `schemaId: $schemaId` from the MATCH pattern.

### C8. String comparison on timestamp in deleteRunsOlderThan
**Where**: `Neo4jExecutionRunRepository.java:112-122`
**Mechanism**: `WHERE r.startedAt < $cutoffTimestamp` — both sides are strings. `$cutoffTimestamp` is `String.valueOf(cutoffTimestamp)` (epoch millis string). String comparison of epoch-millis strings is lexicographic — `"999" > "1000"` is FALSE.
**Impact**: Deletion of old runs is broken — either deletes nothing or deletes wrong runs.
**P**: Certain
**S**: L3 (data cleanup broken, DB grows unbounded)
**R**: **C**
**Fix**: Use numeric comparison or Neo4j `datetime()`.

### C9. Keep-alive caching broken (component names don't match includes)
**Where**: `frontend/src/App.vue:7`, `frontend/src/router/index.ts`
**Mechanism**: `<keep-alive include="dashboard,studio,...">` uses lowercase names. Vue 3 `<script setup>` components get PascalCase names. NO match → StudioView recreated on every navigation.
**Impact**: WebSocket disconnect/reconnect on every schema switch (~3-5s overhead). Schema state re-fetched. Execution progress lost. Exacerbates all other lifecycle bugs (timer leaks, stale state).
**P**: 100%
**S**: L3 (performance degradation, data loss on navigation)
**R**: **C**
**Fix**: Add `defineOptions({ name: 'StudioView' })` to match keep-alive include names.

### C10. Stage timeout doesn't cancel inner NodeRouter execution
**Where**: `PipelineStageRunner.java:144-146` + `NodeRouter.java:222-242`
**Mechanism**: `PipelineStageRunner` wraps `nodeRouter.executeNode()` in CompletableFuture with `.orTimeout(stageTimeoutMs)`. When stage timeout fires: (1) join() throws CompletionException, (2) the inner executionTask CONTINUES running on virtual thread, (3) NodeRouter's inner CompletableFuture is NOT cancelled, (4) results get computed but silently discarded.
**Impact**: Wasted LLM API calls ($$), wasted CPU/file I/O, confusing partial state. Each timeout generates orphan work that runs to completion.
**P**: High (any stage nearing timeout)
**S**: L3 (wasted resources)
**R**: C

## HIGH (R=M-H)

### H5. Cancelled tasks keep spinning on common pool retry loop
**Where**: `NodeRouter.java:212-216`
**Mechanism**: `future.cancel(true)` interrupts thread. Retry loop catches `InterruptedException` from `Thread.sleep()` and restores flag but does NOT break out of loop. Continues spinning through remaining retries.
**Impact**: Each cancelled timeout wastes 5-15 seconds of CPU on common pool thread.
**P**: Medium (every node timeout triggers this)
**S**: L2
**Fix**: Add `if (Thread.currentThread().isInterrupted()) break;` at top of retry loop.

### H6. DB failures logged at DEBUG level with no stack trace
**Where**: `NodeRouter.java:123-125, 238-239, 312-313, 331-332`
**Mechanism**: `log.debug("Не удалось обновить статус узла в БД: {}", e.getMessage())` — all Neo4j persistence failures logged at DEBUG with no stack trace.
**Impact**: DB persistence silently fails. Execution results, status updates lost. Invisible in production logs.
**P**: Medium (any Neo4j connectivity issue)
**S**: L3 (silent data loss)
**Fix**: Log at WARN with full exception.

### H7. Flutter post-processing swallows exceptions with no stack trace
**Where**: `AgentNodeStrategy.java:669-671`
**Mechanism**: Entire Flutter block wrapped in `catch (Exception e) { log.warn("Flutter post-processing failed (non-fatal): {}", e.getMessage()); }` — no stack trace.
**Impact**: Critical bugs invisible in logs.
**P**: Medium
**S**: L2
**Fix**: Log full exception `e`, not just `e.getMessage()`.

### H8. Context compression hard-codes "ollama" model
**Where**: `ExecutionUtilityService.java:207-211`
**Mechanism**: When predecessor results > 4000 chars, calls `llmService.chat("ollama", ...)` — hard-coded provider.
**Impact**: Users on non-Ollama providers hit silent compression failure → truncated context.
**P**: High (non-Ollama users always hit this)
**S**: L2
**Fix**: Use configured model/provider for compression.

### H9. npm shell injection in plugin install (defense depth)
**Where**: `PluginRegistry.java:294-296`
**Mechanism**: `String.format("cd %s/plugins && bun add %s@latest", projectRoot, pkg)` + `ProcessBuilder("bash", "-c", cmd)`. Regex validation exists but bypass → RCE.
**Impact**: RCE if regex bypassed. User-supplied package name in POST /api/open-plugins/install.
**P**: Low (regex is strong)
**S**: L5
**Fix**: Use `ProcessBuilder` list form: `new ProcessBuilder("bun", "add", pkg + "@latest")` with `pb.directory(...)`. Remove shell form entirely.

### H10. JS buffer OOM on parse failure (plugin-bridge.js)
**Where**: `plugin-bridge.js:464-495`
**Mechanism**: Line-by-line accumulation in `buffer`. Single malformed line → buffer grows without bound until OOM.
**Impact**: Bun OOM → plugin killed → restart → repeat. DoS.
**P**: Medium
**S**: L5
**Fix**: Parse each NDJSON line independently. Remove buffer accumulation.

### H11. Hung plugin tool runs forever after timeout
**Where**: `PluginBridge.java:249-259`
**Mechanism**: `future.get(requestTimeoutMs, ...)` times out → `TimeoutException` → entry removed from `pendingRequests` → BUT Bun side keeps executing. No cancel signal sent.
**Impact**: CPU wasted, stale responses arrive as orphans, Bun event loop slows.
**P**: High (any slow tool triggers this)
**S**: L3
**Fix**: Send `plugin/cancel` notification with request ID on timeout. Implement cancel handler in JS.

### H12. Auto-update race vs plugin loading
**Where**: `PluginLifecycleManager.java:113-121`
**Mechanism**: `bun update` runs async in `startupExecutor` AFTER plugins start loading. `import()` hits deleted module files → `MODULE_NOT_FOUND`.
**Impact**: Non-deterministic startup failures. Plugin fails to load.
**P**: Low (race window is narrow)
**S**: L3
**Fix**: Move auto-update to BEFORE plugin initialization, run synchronously.

### H13. No migration framework, stale data accumulates
**Where**: No Liquibase/Flyway/Neo4j-Migrations
**Mechanism**: All schema evolution is ad-hoc (`Neo4jIndexesInitializer`). Stale data from old formats (plan data, string timestamps) never migrated.
**Impact**: Old format data breaks operations silently.
**P**: High
**S**: L3
**Fix**: Add `neo4j-migrations` Spring Boot starter.

### H14. Missing indexes on frequently queried properties
**Where**: All repositories (`NodeExecution.runId`, `Checkpoint.runId`, `ExecutionRecord.schemaId`, `Plan.workspaceId`)
**Mechanism**: No supporting indexes → full label scans on every query. Scales O(n) with history.
**Impact**: Progressive slowdown as data grows. Cleanup queries increasingly expensive.
**P**: High
**S**: L2
**Fix**: Add 5 indexes in `Neo4jIndexesInitializer`.

### H15. Version field dropped in POJO round-trip (locking broken)
**Where**: `ExecutionRepository.java:512-642`
**Mechanism**: `ExecutionRun` POJO has no `version` field. `toGraphRun()`/`toPocoRun()` round-trip drops `@Version`. Optimistic locking ineffective after save→load cycle.
**Impact**: Concurrent writers silently overwrite changes.
**P**: High
**S**: L3
**Fix**: Add version field to POJO and copy in both mapping directions.

### H16. Schema switch leaves stale execution state
**Where**: `StudioView.vue:437-451`
**Mechanism**: `watch(() => route.params.id)` handler never resets `nodeResults`, `nodeStatuses`, `executionProgress`, `stepEvents`, `nodeStartTimes`, `stepCounter`.
**Impact**: Phantom data from previous schema bleeds into new schema UI.
**P**: 100%
**S**: L2
**Fix**: Add reset block after schema switch.

### H17. Static mutable maps in DraftNodeStrategy
**Where**: `DraftNodeStrategy.java:106-120`
**Mechanism**: `private static final Map<String, String>` points to mutable `HashMap`. Any internal code path calling `.put()` corrupts global state for all schemas.
**Impact**: Global shared state corruption affecting all users.
**P**: Low (no current mutation)
**S**: L4
**Fix**: Replace with `Map.of(...)` or `Collections.unmodifiableMap()`.

### H18. GraalVM JS sandbox — no execution timeout
**Where**: `ExecutionUtilityService.java:142-154`
**Mechanism**: GraalVM JS `Context` created with `allowIO(false)` but no time limit. `while(true){}` wedges the thread.
**Impact**: DoS by schema with malicious condition expression.
**P**: Low
**S**: L3
**Fix**: Add `.option("js.timeout", "5000")`.

### H19. No global error handler in frontend
**Where**: `frontend/src/main.ts`, all components
**Mechanism**: No `app.config.errorHandler` or `unhandledrejection` handler. Many `.catch(() => {})` silently swallow errors.
**Impact**: Silent failures. Hard to debug.
**P**: High
**S**: L2
**Fix**: Register global handler. Audit `.catch(() => {})`.
